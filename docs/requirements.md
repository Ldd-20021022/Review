# 需求规格

## 1. 项目定位
多租户 SaaS Web 应用，帮助医院快速完成电子病历系统 4/5/6 级评级的差距自评、整改任务分配与进度跟踪。

## 2. 用户角色

| 角色 | 职责 | 关键权限 |
|---|---|---|
| 超级管理员 | 平台运维 | 租户管理、用户管理、标准库维护（全平台） |
| 评级专家/顾问 | 执行自评 | 指标打分、审核差距项、生成报告 |
| 科室负责人 | 落实整改 | 接收/分配科室任务、更新进度、提交完成 |
| 院领导 | 决策监督 | 查看全院报告和进度看板（只读） |

## 3. 多租户
- 每家医院 = 一个租户，数据完全隔离
- 租户独立管理科室、用户、评估项目
- 同一用户可在不同租户有不同角色

## 4. 评级标准库
- 树形结构：一级分类 → 二级分类 → 指标项
- 每指标含 4/5/6 级要求描述
- 预置标准 + Excel 导入

## 5. 评估项目
- 多项目并行，生命周期：创建 → 自评中 → 审核中 → 整改中 → 已验收 → 归档

## 6. 差距自评
- 百分比打分（0-100%），自动汇总分类得分和综合得分
- 流程：打分 → 标注差距 → 提交 → 复审 → 锁定

## 7. 评估快照
- 锁定时自动生成不可修改快照
- 多版本管理，支持两版对比

## 8. 差距报告 PDF
- 自定义：选择范围/维度/科室
- 在线预览 + 下载

## 9. 整改任务
- 流程：锁定 → 生成任务 → 分配科室 → 执行 → 提交 → 验收
- 状态：待开始 / 进行中 / 已提交 / 已验收 / 退回

## 10. 进度看板
- 全院总览、科室对比、指标维度热力图、时间趋势
